# Code of Conduct

## Our commitment

Alphabet is built as a collaborative, long-lived backend platform. We want this repository to be a respectful, welcoming place for contributors of different backgrounds, experience levels, and communication styles.

By participating in this project, you agree to help create an environment where people can:

- ask questions without embarrassment
- give and receive technical feedback respectfully
- disagree on design decisions without becoming personal
- collaborate across teams and modules without hostility

## Expected behavior

Examples of behavior that strengthen the project and community include:

- being respectful, patient, and professional in issues, pull requests, reviews, and discussions
- explaining technical decisions clearly and constructively
- assuming good intent while still giving honest feedback
- focusing review comments on code, architecture, maintainability, security, and behavior
- helping others understand project conventions such as Clean Architecture boundaries, CQRS, and module ownership
- acknowledging mistakes, correcting them quickly, and learning from them

## Unacceptable behavior

The following behavior is not acceptable in this project:

- harassment, threats, or intimidation
- discriminatory or derogatory language
- personal attacks, mocking, or insulting comments
- hostile review behavior or repeated bad-faith argument
- publishing private or sensitive information without permission
- sexualized language or unwanted attention
- intentionally disruptive behavior in issues, discussions, or pull requests

## Scope

This Code of Conduct applies to:

- repository discussions
- issues and pull requests
- code review comments
- project documentation
- any public or private communication that is clearly connected to this project

## Maintainer responsibilities

Maintainers are responsible for:

- setting clear expectations for collaboration
- enforcing this Code of Conduct consistently and fairly
- de-escalating conflict when possible
- protecting contributors from harassment or abuse
- keeping technical disagreement productive and evidence-based

Maintainers may edit, remove, lock, or reject comments, issues, pull requests, or other contributions that violate this Code of Conduct.

## Reporting concerns

If you experience or witness behavior that violates this Code of Conduct, report it privately to:

- `khanalex301@gmail.com`

Please include as much context as you can, such as links, screenshots, dates, and a short description of what happened. Reports will be handled as discreetly as possible.

## Enforcement

Project maintainers may take any action they consider appropriate, including:

- a private warning
- a request to change behavior
- temporary restriction from project discussions
- rejection of contributions
- temporary or permanent removal from the project community
